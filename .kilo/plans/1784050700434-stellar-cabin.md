# Plan: Close the profile & memory loop — user-configurable at runtime

## Problem
1. Profile adaptations are saved to DuckDB but never merged back into the in-memory `config` dict that services read from.
2. New trades require 10+ similar trades before the LLM sees them (`evidence_min_count=10`).
3. `MemoryWorkspace.trade_count` is never incremented.
4. `min_llm_confidence` threshold is invisible to the LLM.

## Config additions (`config.toml`)

```toml
[adaptive]
auto_merge = true
show_confidence_in_prompt = true

[learning]
evidence_min_count = 1
```

All three settings are editable at runtime via **"View/Edit Configuration"** (config editor). The config editor already serializes nested TOML sections, so no serialization work is needed.

## Tasks (implement in order)

### 1. `src/recommendations/store.py` — add workspace trade counter
Add `increment_workspace_trade_count(workspace_id)`:
```python
self._conn.execute(
    "UPDATE memory_workspaces SET trade_count = trade_count + 1, last_used_at = ? WHERE workspace_id = ?",
    [datetime.utcnow(), workspace_id],
)
```

### 2. `src/learning/pipeline.py` — respect `evidence_min_count` override
In `process_candidate()` (stage 6), before checking `policy.evidence_min_count`, check if `runtime_config_values` contains `learning.evidence_min_count`. If present:
- Value `1` → bypass pending gate, proceed to persistence
- Value `> 1` → use it instead of the policy default

### 3. `src/agent/prompt_compiler.py` — accept `config`, show confidence
- Add `config: dict` parameter to `render()`
- Thread it to `_build_portfolio_section(portfolio, config)`
- When `config.get("adaptive", {}).get("show_confidence_in_prompt", False)`, append:
  ```
  LLM Confidence Threshold: {min_llm_confidence:.2f}
  (Decisions below this threshold are rejected by RiskManager. Calibrate your confidence accordingly.)
  ```

### 4. `src/services/reasoning_coordinator.py` — pass `config` to prompt compiler
- Accept `config` in `evaluate_candidate()`
- Pass it to `prompt_compiler.render()`

### 5. `src/services/trade_coordinator.py` — pass `config`, increment trade count
- Pass `self._config` to `reasoning_coordinator.evaluate_candidate()`
- After `learning_pipeline.process_candidate()` succeeds, call `config_store.increment_workspace_trade_count(active_workspace.workspace_id)`

### 6. `src/main.py` — auto-merge adaptive config in reflection loop
In `_reflection_loop()` after `adaptations = profile_adapter.adapt()`:
```python
if config.get("adaptive", {}).get("auto_merge", True):
    active_adaptations = config_store.get_active_adaptive_versions(active_profile.profile_id)
    param_defs = config_store.get_all_adaptive_parameters()
    if active_adaptations:
        config = merge_adaptive_config(config, active_adaptations, param_defs)
        trade_coordinator._config = config
        risk_mgr._min_llm_confidence = config.get("risk", {}).get("min_llm_confidence", 0.3)
        execution_svc._config = config
```
In `_on_position_closed_learning()`, after successful pipeline processing:
```python
config_store.increment_workspace_trade_count(active_workspace.workspace_id)
```

### 7. `src/cli/config_editor.py` — expose new settings
Add to `_OPERATIONAL_PARAMS`:
- `("adaptive.auto_merge", "Auto-Merge Adaptive Config", "RiskManager", "Re-merge profile adaptations into runtime config after each reflection cycle", "bool", None, None, True)`
- `("adaptive.show_confidence_in_prompt", "Show Confidence in Prompt", "LLMRegistry", "Show min_llm_confidence threshold in the LLM prompt for self-calibration", "bool", None, None, True)`
- `("learning.evidence_min_count", "Evidence Min Count", "LearningPipeline", "Minimum evidence count before a trade is finalized as an experience (1 = immediate)", "int", 1, 30, 1)`

Add "Adaptive & Learning" to `_SECTION_ORDER` and `_SECTION_COMPONENT_MAP`.

### 8. `config.toml` — add the two new sections

## Runtime propagation approach
**In-place instance variable mutation.** After each reflection cycle, update `trade_coordinator._config`, `risk_mgr._min_llm_confidence`, and `execution_svc._config` directly. This is the minimal-change approach: the reflection loop already holds references to all service objects, the mutation is explicit and easy to trace in logs, and it avoids introducing a shared config provider that would touch every service constructor.

## Validation
1. Start APEX, confirm the "View/Edit Configuration" menu shows the three new settings under "Adaptive & Learning"
2. Change `evidence_min_count` from 1 to 5 in the editor, save, restart, and verify pending candidates accumulate until 5 similar trades
3. Change it back to 1, close one trade, and verify retrieval finds it immediately (no cold-start message)
4. Set `show_confidence_in_prompt = true`, run a candidate, and confirm the prompt includes the confidence threshold text
5. Set `auto_merge = true`, let the reflection loop run, and verify the log shows "Config merged from adaptive version" when beliefs trigger adaptations
6. Verify `trade_count` increments in the Memory Workspaces table after a position closes
